# Часть 1. Проектирование и анализ: Генератор QR-кодов

## 1.1. Пользовательские сценарии (User Stories)
1. **Создание сложных QR-кодов:** Как профессиональный пользователь, я хочу иметь возможность легко сгенерировать QR-код для подключения к Wi-Fi или сохранения контакта (vCard), заполнив понятную форму вместо ручного ввода спецсимволов.
2. **Глубокая кастомизация (Превью):** Как дизайнер, я хочу тонко настраивать внешний вид QR-кода (градиенты, форма точек, выбор уровня коррекции ошибок, загрузка логотипа по центру) и моментально видеть превью, чтобы QR-код соответствовал брендингу компании.
3. **Управление историей:** Как пользователь, я хочу, чтобы система запоминала мои удачные генерации, чтобы я мог в любой момент загрузить их снова, скачать в PNG/SVG или удалить из истории неактуальные черновики.

## 1.2. Функциональные требования
**Обязательные (MVP):**
* Фронтенд:
  * Вкладки с формами (URL, Текст, Wi-Fi параметры, vCard).
  * Панель премиум-настроек (Цвета, градиенты линейные/радиальные, форма точек/углов, Error Correction Level - H/Q/M/L).
  * Дебаунс (Debounce) динамического превью на Canvas при изменении настроек.
  * Кнопки скачивания (PNG/SVG) и кнопка сохранения в БД.
  * Список-лента с историей кодов и кнопками их загрузки/удаления.
* Бэкенд:
  * REST API (Express) для получения истории, добавления новых записей и их удаления.
  * Хранение готовых DataURL-изображений (и Base64 логотипов) в БД.

**Опциональные (Продуктовые фичи):**
* Переключатель тем оформления "Светлая/Тёмная".
* Предустановленные стили "Пресеты" (Neon, Classic, Soft) для быстрого старта.

## 1.3. Проектирование API
Бэкенд будет предоставлять следующие endpoint'ы:

1. `POST /api/qr/generate`
   * **Описание:** Генерирует QR-код, сохраняет его в историю и возвращает данные.
   * **Body (JSON):** 
     ```json
     {
       "content": "https://example.com",
       "colorDark": "#000000",
       "colorLight": "#ffffff",
       "size": 300
     }
     ```
   * **Response (200 OK):**
     ```json
     {
       "id": "uuid-string",
       "content": "Внутренняя строка данных QR",
       "qrCode": "data:image/png;base64,...",
       "logoBase64": "data:image/png;base64,... (optional)",
       "createdAt": "2026-03-09T12:00:00Z"
       // + мета-поля стилей
     }
     ```

## 1.4. Модель данных
Для хранения истории используется легковесная файловая БД в формате JSON (`db.json`). Из-за небольших нагрузок это является отличным решением и исключает сложные процессы компиляции C-зависимостей (`node-gyp`).

**Структура JSON-объекта `HistoryItem`**
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | UUID | Уникальный ключ записи |
| `content` | String | Содержимое QR (например, Ссылка или строка `WIFI:T:...`) |
| `colorDark` / `colorLight` | Hex | Настройки цвета |
| `dotsType` / `corners*` | String | Маркеры форм (rounded, dots, classy и т.д.) |
| `logoBase64` | String | Картинка логотипа в Base64 (опционально) |
| `qrCode` | String | Изображение финального QR-кода в формате DataURL |
| `createdAt` | DateTime | Дата создания |

## 1.5. Ключевые технические решения (Стек)
* **Frontend:** `React` (на базе Vite 7) + `Tailwind CSS v4`.
  * *Почему:* Быстрый запуск, модульность и супер-надежное превью на Canvas. Tailwind v4 избавляет от громоздкой настройки webpack/postcss конфигураций.
  * *Библиотеки:* `qr-code-styling` (профессиональный рендеринг QR на Canvas с поддержкой градиентов, форм, скачивания в SVG/PNG "из коробки" прямо на клиенте). 
* **Backend:** `Node.js` + `Express`.
  * *Почему:* Код един на обеих сторонах (JS). Легко настроить REST, нет кроссплатформенных препятствий.
  * *Библиотеки:* Файловая система (FS) для парсинга `db.json`. Валидация входных данных: `zod`, генерация ключей: `uuid`. Совместный запуск: `concurrently`.
* **База Данных:** `JSON File Storage (db.json)`.
  * *Почему:* Не требует установки сервера БД, нет проблем с зависимостями (`sqlite3` часто падает при установке в Windows). Идеально подходит для MVP и демонстрации работы CRUD.
